System Architecture Design
1. User Layer
Programme actors interacting with the platform at different control levels.
TIER 1-3

Executive and Admin Control

  • System administrators and PMO leads
  • Configuration Panel ownership
  • Financial, governance, and HR oversight
TIER 4-7

Operational Users

  • Contract, programme, vendor, and HR users
  • Restricted workflows and dashboard access
  • Role-based approval and data-entry permissions
2. Capture & Workflow Layer
Operational forms, submissions, approvals, and import channels.
FORMS

Programme Transactions

  • Milestones, claims, contracts, meetings, HR records
  • Mandatory fields and attachment gates
  • Approval-driven status transitions
IMPORTS

FDT Integration

  • CSV batch imports and validation
  • Deduplication and discrepancy handling
  • Immutable import event records
3. Core Platform Layer
ERPNext/Frappe services for data, business logic, and governance controls.
APPLICATION

AIPMS Core

  • Milestone, contract, payment, document, governance, reporting, and HR modules
  • Configuration-driven master data and programme parameters
WORKFLOWS

Control Engine

  • RBAC, approvals, alerts, and escalation routing
  • Retention, payment certification, and reconciliation logic
4. Integration Layer
External interfaces and technical services connected to AIPMS.
DATA

External Inputs

  • FDT batch files and optional API integration
  • Email notifications and export delivery channels
PLATFORM

Infrastructure Services

  • Database, backups, audit logging, and storage
  • Security, encryption, and monitoring
5. Output & Oversight Layer
Decision support, auditability, and management visibility.
DASHBOARDS

Operational Intelligence

  • Governance, PMO, finance, HR, escalation, and import log dashboards
  • Role-appropriate reporting and export outputs
ASSURANCE

Audit & Compliance

  • Immutable logs, document history, and approved-record control
  • Read-only audit access and transition-ready exports
End-to-End Process Flow
Programme Parameters Configured
→
Operational Records and FDT Data Captured
→
Workflow, Validation, and Approval Controls Applied
→
Dashboards, Certificates, Reports, and Audit Outputs Generated
