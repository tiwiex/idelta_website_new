DIGISTEM System Architecture

Digital Street Enumeration & Management System

1. User Layer
Primary actors interacting with the system
FIELD USERS

Enumerators

  • Verify streets on the ground
  • Capture house counts
  • Classify buildings
  • Upload photos
  • Capture GPS coordinates
SUPERVISION

Supervisors

  • Review submitted records
  • Validate field entries
  • Monitor enumerator activity
  • Approve data for back-office processing
ADMINISTRATION

Back Office / Admin

  • Clean and validate data
  • Manage street naming workflow
  • Plan signage and numbering
  • Track project implementation
2. Field Capture Layer
Web-based interface used by field teams
PWA

Progressive Web Application

  • Responsive interface for phones and tablets
  • Street verification form
  • Building enumeration form
  • Offline-first or low-connectivity support
  • Sync to central platform
DATA CAPTURE

Captured Information

  • Street existence confirmation
  • Street type: local, collector, arterial
  • Buildings left/right by entry direction
  • Residential / commercial / open space
  • Commercial sub-type details
MEDIA & LOCATION

Evidence Collection

  • Photo attachments
  • Geotagging
  • Timestamp logging
  • Enumerator activity trace
3. Core Platform Layer
Central system for workflows, records and reporting
APPLICATION CORE

DIGISTEM Platform

  • Street registry
  • Building registry
  • Enumeration sessions
  • Photo and evidence storage references
  • Validation workflows
BUSINESS LOGIC

Workflow Engine

  • Submission and review workflow
  • Supervisor approval
  • Data correction and resubmission
  • Status tracking
  • Implementation progress updates
DATABASE

Central Data Store

  • Street master data
  • Building and unit records
  • GPS coordinates
  • Photo metadata
  • Address and signage plans
4. Integration Layer
External services and data sources connected to the platform
GIS

GIS / Shapefile Integration

  • Import existing street geometries
  • Match field records to map features
  • Expose street boundaries and routes
MEDIA STORAGE

File / Image Storage

  • Store photos and documents
  • Link media to street and building records
  • Support secure access
LOCATION SERVICES

Geolocation Services

  • Browser/device GPS capture
  • Coordinate validation
  • Map display support
5. Output & Management Layer
Decision support, planning and implementation tracking
QUALITY CONTROL

Validation & Review

  • Data cleansing
  • Exception handling
  • Approval dashboard
  • Record completeness checks
PLANNING

Address Management

  • Street naming workflow
  • House numbering plan
  • Signage placement planning
REPORTING

Dashboards & Reports

  • Enumeration progress
  • Validated street totals
  • Installation progress tracking
  • Operational performance metrics

End-to-End Process Flow

Existing Street / GIS Data Loaded
→
Enumerator Captures Field Data in PWA
→
Data Syncs to Central Platform
→
Supervisor / Admin Validates Records
→
Street Naming & House Number Planning
→
Signage / Numbering Implementation Tracking

Architecture Notes

  • The field layer is designed as a Progressive Web Application (PWA), so enumerators use a browser-based interface rather than a native mobile app.
  • The core platform can be implemented with an ERP/workflow backend that manages structured data, approvals, reporting and administration.
  • GIS integration supports reuse of existing shapefiles and mapping datasets.
  • The system supports a full lifecycle approach: enumeration, validation, address planning, signage deployment, and reporting.
  • The design is suitable for phased delivery, starting from business analysis and data capture, then extending into project tracking and implementation management.
