FEVA System Architecture

Digital Street Enumeration & Management System
1. Architecture Overview
User Layer Field Capture Layer Core Platform Layer Integration Layer Output & Management Layer
USERS
Enumerators
  • Verify streets on the ground
  • Count houses and structures
  • Classify building types
  • Capture photos and location data
Supervisors
  • Review submissions
  • Monitor field progress
  • Approve or return records
Administrators
  • Manage master data
  • Control workflows
  • Plan implementation activities
PWA
Progressive Web Interface
  • Browser-based field capture
  • Street verification forms
  • Building enumeration forms
  • Low-connectivity support
  • Secure synchronization
Captured Data
  • Street existence and type
  • Left/right building counts
  • Residential / commercial / open space
  • Commercial category details
  • Photo and GPS evidence
CORE PLATFORM
FEVA Application
  • Street registry
  • Building registry
  • Enumeration sessions
  • Review and approval workflows
  • Central record management
Central Database
  • Street records
  • Building records
  • GPS coordinates
  • Photo references
  • Address planning data
INTEGRATIONS
GIS / Shapefile Data
  • Import existing street geometry
  • Link records to mapped streets
  • Support spatial validation
Supporting Services
  • Photo / file storage
  • Geolocation services
  • API-based data exchange
OUTPUTS
Validation & Quality Control
  • Record review and cleaning
  • Exception handling
  • Approval dashboards
Planning & Tracking
  • Street naming workflow
  • House numbering plan
  • Street sign planning
  • Implementation progress tracking
Reporting
  • Enumeration progress reports
  • Street totals and validation status
  • Deployment dashboards
2. End-to-End Process Flow
Existing GIS / Street Data Loaded → Enumerator Captures Field Data → Data Syncs to Central Platform → Supervisor / Admin Validation → Address Planning & Numbering → Signage / Numbering Tracking
3. Architecture Notes
  • The field capture interface is designed as a Progressive Web Application (PWA), allowing enumerators to use a browser-based interface instead of a native mobile application.
  • The platform supports the full operating lifecycle: enumeration, validation, address planning, signage deployment, and reporting.
  • GIS integration allows reuse of existing street and map datasets.
  • The central application layer supports workflow management, approval processes, dashboards, and controlled data administration.
  • The architecture is suitable for phased delivery, starting with business analysis and platform setup, followed by field capture and implementation tracking.
